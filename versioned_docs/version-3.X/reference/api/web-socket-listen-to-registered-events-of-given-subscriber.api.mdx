## WebSocket Endpoint

The client needs to send all the required headers for a ws connection and has to handle the websocket protocol.<br/> **Note this can not be tested from within swagger!**

The connection to this endpoint will never be closed. All action events of the system this subscriber has registerd will be send to this connection.
